## 一、问题简述

我们使用贵司 API 文档中的**授权令牌**接口进行登录，但无论使用正确的官网登录账号和密码，接口均返回错误码，无法获取 token，导致后续所有 API 无法调用。

## 二、请客服确认或提供以下信息

### 1. 授权令牌接口

- **文档中的接口**：`POST https://blurpath.com/api/api/supplier/accountLogin`  
  请求体：`{"username": "邮箱", "password": "密码"}`

- **当前现象**：
  - 使用与官网登录一致的邮箱和密码调用上述接口；
  - 返回：`{"ret": 1, "msg": ""}`（无 token）；
  - 尝试文档中可能存在的 `passwordLogin` 端点时，返回：`{"ret": 2, "msg": ""}`。

- **需要确认**：
  1. **`ret` 的含义**：`ret: 0` 是否表示成功？`ret: 1`、`ret: 2` 分别代表什么错误？
  2. **`msg` 为空**：为何错误时 `msg` 为空，是否有错误码对照表或文档说明？
  3. **API 登录与官网登录是否一致**：API 的 `accountLogin` 是否必须使用**主账号的邮箱+密码**，还是必须使用**账密授权中创建的子账号**？
  4. **是否有其他登录方式**：例如 API Key、固定 token、或其它认证方式（若账号密码登录已废弃或仅限部分账户）。

### 2. 账号与权限

- **需要确认**：
  1. 使用**主账号**（注册邮箱+密码）调用 `accountLogin` 获取 token 是否被支持？若支持，需要满足哪些条件（例如：已购产品、账户状态、地区限制等）？
  2. 若必须使用**子账号**：子账号是在「账户管理 → 账密授权 → 添加账户」中创建的吗？创建后是否用该子账号的「用户名+密码」调用 `accountLogin`？请求体字段是否仍为 `username` 和 `password`？
  3. 调用 API 前是否需要在控制台**单独开通或激活 API 权限**？如有，请说明入口和步骤。

### 3. 接口与文档

- **需要确认或提供**：
  1. **当前有效的授权接口地址**（若与文档不一致）：例如是否仍为 `https://blurpath.com/api/api/supplier/accountLogin`，或有新的 base URL / 路径？
  2. **请求格式**：Content-Type 是否为 `application/json`？请求体是否必须为 `username` + `password`，是否有额外必填参数（如 `captcha`、`clientId` 等）？
  3. **成功响应示例**：请提供一个**成功获取 token** 的响应样例（可打码敏感信息），便于我们对照解析。
  4. **错误码文档**：是否有 `ret`、`msg` 或其它错误码的官方说明文档或表格？若有，请提供链接或附件。

### 4. 环境与限制

- **需要确认**：
  1. API 是否对**调用 IP** 或**地区**有限制（例如仅允许某些国家/地区或需要先加 IP 白名单才能登录）？
  2. 是否存在**频率限制**（如每分钟/每小时调用次数），以及触发限制时的返回格式？
  3. 是否需要**先购买具体产品**（如静态 ISP、动态住宅等）才能使用授权接口获取 token？
