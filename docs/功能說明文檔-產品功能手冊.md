## 1. 產品概述

### 1.1 產品定位

**TG-AI智控王** 是專為 Telegram 生態設計的 **AI 驅動智能營銷自動化平台**，幫助用戶實現：

- 多帳號矩陣管理
- 群組監控與用戶收集
- AI 智能對話與多角色協作
- 全鏈路自動化營銷
- 數據分析與轉化追蹤

### 1.2 核心價值

| 痛點 | 解決方案 |
|------|----------|
| 手動添加客戶效率低 | AI 自動識別潛在客戶，批量收集 |
| 單一帳號容易封禁 | 多帳號矩陣分散風險 |
| 人工回覆不及時 | AI 7×24 小時即時響應 |
| 銷售話術不專業 | AI 智能生成專業話術 |
| 轉化率難以提升 | 多角色協作提升信任 |
| 數據分析困難 | 實時數據看板追蹤 |

## 2. 核心功能模塊

### 2.1 儀表板 (Dashboard)

**路徑**: `/dashboard`

**功能說明**  
系統首頁，集中展示運行狀態與關鍵指標。

**主要內容**：
- 帳號狀態卡片（在線/離線/連接中）
- 今日統計：發送/接收消息數、AI 回覆次數、新增線索
- 快捷操作：一鍵啟動監控、快速發送
- 最近活動記錄

---

### 2.2 帳號管理 (Accounts)

**路徑**: `/accounts`

**功能說明**  
管理多個 Telegram 帳號，支持多種登錄方式。

**子功能**：
| 功能 | 說明 |
|------|------|
| 帳號列表 | 查看所有帳號及狀態（在線/離線/連接中/受限/封禁） |
| 添加帳號 | QR 碼 / 手機號 / Session 文件導入 |
| API 憑證 | 管理 Telegram API ID / Hash |
| 批量操作 | 批量刷新、刪除、導出 |

**帳號狀態**：
- 🟢 在線：帳號已連接可用
- 🔴 離線：未連接
- 🟡 連接中：正在嘗試連接
- 🟠 受限：帳號被限制
- ⚫ 封禁：帳號已被封禁

---

### 2.3 監控中心 (Monitoring)

**路徑**: `/monitoring`

**功能說明**  
監控 Telegram 群組，通過關鍵詞與規則觸發自動化。

**子功能**：
| 功能 | 說明 |
|------|------|
| 監控群組 | 添加/管理待監控群組，查看監控狀態 |
| 關鍵詞集 | 配置觸發關鍵詞，支持正則 |
| 觸發規則 | 設置觸發條件與動作（發送消息、收集用戶等） |
| 聊天模板 | 配置自動回覆模板 |
| 用戶收集 | 從群組歷史消息收集用戶（含進階篩選） |

**v2.1.0 用戶收集增強**：
- 時間範圍：7 天 / 30 天 / 90 天 / 全部
- 活躍度：發言次數 ≥1/3/5/10
- 快速模板：快速收集 / 精準收集 / 深度收集
- 進階選項：去重、排除機器人、需用戶名等
- 實時進度與質量統計

---

### 2.4 潛在客戶 (Leads)

**路徑**: `/leads`

**功能說明**  
管理從群組和私聊中收集的潛在客戶，支持標籤與跟進。

**子功能**：
- 線索列表與篩選
- 狀態流轉：新發現 → 已聯繫 → 有興趣 → 談判中 → 已成交 / 已流失
- 批量操作：標記、分配、導出
- 跟進記錄與備註
- 與多角色協作聯動

---

### 2.5 自動化中心 (Automation)

**路徑**: `/automation`

**功能說明**  
配置自動化活動、模板與觸發鏈。

**子功能**：
- 活動管理：創建、啟用/暫停活動
- 模板管理：話術模板、變量佔位符
- 觸發鏈：監控 → 匹配 → 發送 → 跟進的完整流程
- 消息隊列：發送隊列、限流與重試

## 3. 進階功能模塊

### 3.1 營銷任務中心 (Marketing Hub)

**路徑**: `/marketing-hub`

**功能說明**  
營銷任務的統一入口，整合 AI 策劃、角色協作與執行監控。

**子功能**：
- 營銷任務創建與配置
- 目標設定：促進成交、品牌推廣、用戶培育等
- AI 智能規劃：角色推薦、話術生成
- 任務執行監控與日誌
- 轉化漏斗與效果分析

---

### 3.2 角色資源庫 (Role Library)

**路徑**: `/role-library`

**功能說明**  
管理多角色協作所需的人物與腳本。

**子功能**：
- 角色模板：成交專家、產品專家、客服、見證者等 50+ 模板
- 自定義角色：設定話術風格與職責
- 腳本編排：多角色協作流程
- 場景模板：不同營銷場景的預設配置

---

### 3.3 智能引擎 (AI Engine)

**路徑**: `/ai-engine`

**功能說明**  
配置 AI 模型、知識庫與對話策略。

**子功能**：
| 功能 | 說明 |
|------|------|
| 模型配置 | API Key、模型選擇、參數調優 |
| 知識庫 | RAG 知識增強，上傳文檔/鏈接 |
| 意圖識別 | 用戶意圖分類與路由 |
| 話術生成 | 根據場景生成營銷話術 |
| AI 文案 | 標題、正文、結尾等批量生成 |

---

### 3.4 資源發現 (Resource Discovery)

**路徑**: `/resource-discovery`

**功能說明**  
搜索與發現 Telegram 群組資源，並提取成員。

**子功能**：
- 群組搜索：關鍵詞、類別篩選
- 群組詳情：成員數、活躍度等
- 成員提取：批量導出群成員
- 加入/監控群組

---

### 3.5 數據分析 (Analytics)

**路徑**: `/analytics`

**功能說明**  
查看營銷效果與轉化數據。

**子功能**：
- 轉化漏斗：各階段轉化率
- 趨勢圖表：發送量、回覆量、轉化趨勢
- 角色效能：各角色貢獻度
- ROI 分析
- 報表導出

---

### 3.6 錢包系統 (Wallet)

**路徑**: `/wallet` 及子路徑

**功能說明**  
用戶餘額、充值、提現與訂單管理。

**子功能**：
| 路徑 | 說明 |
|------|------|
| /wallet | 錢包總覽、餘額 |
| /wallet/recharge | 充值中心 |
| /wallet/withdraw | 提現 |
| /wallet/transactions | 交易記錄 |
| /wallet/orders | 充值訂單 |

## 6. 使用流程概覽

### 6.1 典型營銷流程

```
1. 添加帳號 → 2. 配置監控群組與關鍵詞
     ↓
3. 設置觸發規則與模板 → 4. 開啟自動化
     ↓
5. 用戶收集 → 6. 潛在客戶管理
     ↓
7. 多角色協作 / 營銷任務 → 8. 數據分析與優化
```

### 6.2 首次使用建議

1. 添加至少一個 Telegram 帳號
2. 配置 API 憑證（若未使用默認）
3. 添加監控群組並設置關鍵詞
4. 配置觸發規則與回覆模板
5. 開啟監控，觀察數據與效果
6. 根據需要啟用 AI 與多角色協作
