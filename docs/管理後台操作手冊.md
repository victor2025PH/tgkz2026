## 登錄與安全

### 訪問地址

```
https://tgkz.usdt2026.cc/admin/login.html
```

### 默認帳號

| 項目 | 值 |
|------|-----|
| 用戶名 | admin |
| 密碼 | admin888 |

⚠️ **首次登錄後請立即修改密碼！**

### 登錄安全

- **失敗鎖定**: 連續 5 次密碼錯誤，帳號鎖定 15 分鐘
- **Session**: 登錄有效期 24 小時
- **IP 記錄**: 每次登錄記錄 IP 地址

### 退出登錄

點擊右上角用戶名 → 選擇「退出登錄」

## 儀表盤

### 功能說明

儀表盤是進入後台後的首頁，顯示系統整體運營數據。

### 數據卡片

| 卡片 | 說明 |
|------|------|
| 總用戶數 | 系統註冊的總用戶數 |
| 今日新增 | 今天新註冊的用戶數 |
| 活躍用戶 | 近 24 小時有心跳的用戶 |
| 總收入 | 所有已支付訂單的總金額 |
| 今日收入 | 今天的已支付訂單金額 |
| 總卡密 | 系統生成的卡密總數 |
| 未使用卡密 | 還未被激活的卡密數量 |
| 待處理訂單 | 待支付確認的訂單數 |

### 圖表

- **等級分布圓餅圖**: 各會員等級的用戶數量佔比

### 刷新數據

點擊右上角的「刷新」按鈕可手動刷新數據。

## 用戶管理

### 訪問方式

側邊欄 → 👥 用戶管理

### 用戶列表

顯示所有註冊用戶，包含以下信息：

| 欄位 | 說明 |
|------|------|
| 用戶 ID | 唯一標識符 |
| Email | 用戶郵箱 |
| 等級 | 會員等級（顯示圖標和名稱） |
| 過期時間 | 會員到期日期 |
| 消費總額 | 用戶累計消費金額 |
| 邀請人數 | 該用戶邀請的人數 |
| 狀態 | 正常/已封禁 |
| 最後活躍 | 最後一次心跳時間 |

### 搜索和篩選

- **搜索框**: 按用戶 ID、郵箱搜索
- **等級篩選**: 篩選特定等級的用戶

### 查看用戶詳情

點擊用戶行的「詳情」按鈕，彈出用戶詳情窗口：

**基本信息區域**:
- 用戶 ID
- 郵箱
- 暱稱
- 邀請碼
- 機器碼
- 註冊時間

**會員信息區域**:
- 當前等級
- 過期時間
- 剩餘天數
- 是否終身

**消費統計**:
- 總消費
- 邀請收益
- 餘額

**配額使用**:
- 各項配額的使用情況

**卡密記錄**:
- 該用戶使用過的卡密列表

### 手動續費

1. 點擊用戶行的「續費」按鈕
2. 在彈窗中輸入天數（1-365）
3. 選擇原因（可選）
4. 點擊確認

### 封禁用戶

1. 點擊用戶行的「封禁」按鈕
2. 在確認對話框中輸入封禁原因
3. 確認封禁

**封禁效果**:
- 用戶無法登錄
- 現有 Token 失效
- 可隨時解封

### 解封用戶

對已封禁的用戶，點擊「解封」按鈕即可恢復。

## 卡密管理

### 訪問方式

側邊欄 → 🎟️ 卡密管理

### 卡密列表

顯示所有生成的卡密：

| 欄位 | 說明 |
|------|------|
| 卡密 | 完整卡密碼 |
| 等級 | 對應的會員等級 |
| 時長 | 周/月/季/年/終身 |
| 天數 | 具體天數 |
| 價格 | 卡密定價 |
| 狀態 | 未使用/已使用/已禁用 |
| 使用者 | 激活該卡密的用戶 |
| 創建時間 | 卡密生成時間 |

### 篩選卡密

- **狀態篩選**: 全部/未使用/已使用/已禁用
- **等級篩選**: 篩選特定等級的卡密

### 批量生成卡密

1. 點擊「生成卡密」按鈕
2. 選擇會員等級（白銀/黃金/鑽石/星耀/王者）
3. 選擇時長（周卡/月卡/季卡/年卡/終身）
4. 輸入數量（1-100）
5. 可選：輸入批次名稱和備註
6. 點擊「生成」

**注意**: 
- 每次最多生成 100 個
- 生成後會顯示卡密列表，可複製或導出

### 禁用卡密

1. 點擊卡密行的「禁用」按鈕
2. 確認禁用

**禁用效果**:
- 卡密無法被激活
- 已激活的卡密不受影響

### 導出卡密

1. 點擊「導出」按鈕
2. 選擇導出範圍（全部/未使用/篩選結果）
3. 下載 CSV 文件

## 訂單管理

### 訪問方式

側邊欄 → 💰 訂單管理

### 訂單列表

| 欄位 | 說明 |
|------|------|
| 訂單號 | 唯一訂單標識 |
| 用戶 | 下單用戶 |
| 產品 | 購買的產品名稱 |
| 金額 | 訂單金額 |
| 狀態 | pending/paid/cancelled |
| 創建時間 | 下單時間 |
| 支付時間 | 支付完成時間 |

### 訂單狀態說明

| 狀態 | 說明 | 顏色 |
|------|------|------|
| pending | 待支付 | 黃色 |
| paid | 已支付 | 綠色 |
| cancelled | 已取消 | 灰色 |

### 手動確認支付

當用戶 USDT 支付後未自動確認時：

1. 找到對應訂單
2. 點擊「確認支付」按鈕
3. 確認操作

**確認後**:
- 訂單狀態變為 paid
- 自動為用戶激活會員
- 生成對應卡密記錄
- 觸發邀請獎勵（如有）

## 收入報表

### 訪問方式

側邊欄 → 💹 收入報表

### 時間範圍

點擊頂部按鈕切換時間範圍：
- **7天**: 近一週數據
- **30天**: 近一月數據
- **90天**: 近三月數據

### 概覽卡片

| 卡片 | 說明 |
|------|------|
| 總收入 | 所選時間段的總收入 |
| 增長率 | 與上一時間段對比的增長百分比 |
| 訂單數 | 已支付訂單數量 |
| 付費用戶 | 獨立付費用戶數 |

### 圖表

#### 收入趨勢圖（柱狀圖）
- 橫軸: 日期
- 縱軸: 收入金額
- 顯示每日收入變化趨勢

#### 等級收入分布（餅圖）
- 顯示各等級的收入佔比
- 幫助了解哪個等級最受歡迎

### 詳細表格

#### 按等級統計
- 每個等級的訂單數和收入

#### 按時長統計
- 每種時長（周/月/季/年/終身）的訂單數和收入

## 公告管理

### 訪問方式

側邊欄 → 📢 公告管理

### 公告列表

| 欄位 | 說明 |
|------|------|
| 標題 | 公告標題 |
| 類型 | system/promotion/update/other |
| 置頂 | 是否置頂顯示 |
| 彈窗 | 是否彈窗顯示 |
| 狀態 | draft/published |
| 創建時間 | 發布時間 |

### 發布新公告

1. 點擊「發布公告」按鈕
2. 填寫公告信息：
   - **標題**: 公告標題（必填）
   - **內容**: 公告正文（必填）
   - **類型**: 系統/活動/更新/其他
   - **優先級**: 數字越大越靠前
   - **置頂**: 是否置頂
   - **彈窗**: 是否在客戶端彈窗顯示
   - **過期時間**: 可選，設置後到期自動隱藏
3. 點擊「發布」

### 編輯公告

1. 點擊公告行的「編輯」按鈕
2. 修改內容
3. 點擊「保存」

### 刪除公告

1. 點擊公告行的「刪除」按鈕
2. 確認刪除

## 常見操作流程

### 流程1: 新用戶開通會員

1. 進入「卡密管理」
2. 點擊「生成卡密」
3. 選擇等級和時長
4. 生成卡密
5. 將卡密發送給用戶

### 流程2: 處理 USDT 支付

1. 用戶反饋已支付但未激活
2. 進入「訂單管理」
3. 找到對應訂單
4. 核實用戶提供的交易哈希
5. 點擊「確認支付」

### 流程3: 處理用戶投訴

1. 進入「用戶管理」
2. 搜索用戶
3. 查看用戶詳情
4. 根據情況：手動續費/修改等級/聯繫用戶

### 流程4: 查看運營數據

1. 每日查看「儀表盤」了解整體情況
2. 每週查看「收入報表」分析收入趨勢
3. 每月查看「用戶分析」評估用戶健康度

## 故障排除

### Q1: 登錄失敗

1. 檢查用戶名和密碼是否正確
2. 如帳號鎖定，等待 15 分鐘
3. 清除瀏覽器緩存重試

### Q2: 頁面加載緩慢

1. 檢查網絡連接
2. 刷新頁面
3. 聯繫技術支持檢查服務器

### Q3: 數據不更新

1. 點擊刷新按鈕
2. 退出登錄後重新登錄
3. 清除瀏覽器緩存
